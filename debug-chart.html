Focus Chart Debugging

🔍 Debugging Instructions

  1. Open the Smart Bookmarks extension popup
  2. Go to the Focus tab
  3. Open Developer Tools (F12)
  4. Look for console logs starting with 🎯, 📊, ✅, or ❌
  5. Check the logs below to understand the issue

📋 Expected Console Logs

🎯 updateFocusChart called Chart containers found: { focusChart: false, focusChartInsight: true } 📊 User data: { todayFocusTime: 6, completedSessions: 2, ... } 📅 Focus sessions: [...] ⏱️ Today focus time: 6 📊 Last 7 days data: [0, 0, 0, 0, 0, 0, 6] 📅 Day labels: ["Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun"] 🧪 Using TEST DATA for debugging 📊 Generated chart HTML length: 300+ ✅ Updating focusChartInsight container ✅ focusChartInsight container updated, children: 1 🎯 updateFocusChart completed

🐛 Common Issues

Issue Cause Solution
No containers found DOM not ready or wrong tab active Ensure Focus tab is active
Chart HTML empty Data processing error Check test data is enabled
No children after update HTML injection failed Check for CSP violations

🔧 Quick Fixes to Try

  1. Force Refresh: Close and reopen the extension popup
  2. Clear Storage: Reset data in Settings tab
  3. Check Active Tab: Make sure you're on the Focus tab
  4. Test Mode: The code has test data enabled (line 816: useTestData = true)
  5. Fallback Chart: After 100ms, a fallback chart should appear if main chart fails
